# Общее ТЗ (MVP) — Hemvision: веб-приложение для анализа и шаринга морфологии лейкоцитов

## 1. Цели и назначение
Hemvision — веб-приложение для сотрудников гематолабораторий и студентов, позволяющее:
- загружать изображения/видео мазков крови;
- выполнять автоматический детект лейкоцитов (bbox, class, score) на CPU;
- вручную корректировать классы;
- объединять медиа в «Анализы», считать сводку по классам, экспортировать в CSV;
- делиться результатами по защищённым ссылкам (read-only) с настраиваемым сроком жизни.

MVP ориентирован на развёртывание на собственном сервере Заказчика.

## 2. Глоссарий
- **Медиа** — изображение (PNG/JPG) или видео (MP4/AVI ≤ 20 MB).
- **Детекция** — результат модели: bbox (x,y,w,h), class, score; для видео — с привязкой к кадру.
- **Класс** — категория клетки (базовые + пользовательские).
- **Анализ** — именованный набор медиа с агрегированными подсчётами по классам.
- **Share-ссылка** — публичный read-only доступ к медиа или анализу по токену и TTL.

## 3. Аудитория и роли
- **OrgAdmin** — администратор организации: управление участниками, классами, данными.
- **Member** — сотрудник/студент в составе организации: загрузка, инференс, правки, анализы.
- **Individual** — личный аккаунт без организации (тот же функционал, но в личном скоупе).
- **Viewer (по ссылке)** — гость по Share-ссылке: только просмотр.

## 4. Область охвата (Scope)
### Включено (MVP)
- Регистрация/логин по email+пароль (Argon2), JWT-сессии.
- Организации по подписке (биллинг вне MVP, хранится только seats).
- Базовые классы: neutrophil, lymphocyte, monocyte, eosinophil, basophil, blast.
- Пользовательские классы на уровне организации.
- Загрузка медиа в MinIO (presigned URL), хранение метаданных (окраска, увеличение и т.п.).
- Инференс YOLOv11 (ONNX Runtime CPU):
  - изображения: ≤ 1 с целевое время (при фиксированном input size);
  - видео: простой режим — сэмплинг **1 кадр/сек, максимум 10 кадров/видео**.
- Просмотрщик с overlay bbox и сменой класса (orig_class сохраняется).
- Анализы: формирование из выбранных медиа, сводные подсчёты по классам.
- Экспорт CSV (агрегат + опционально детализация по детекциям).
- Шаринг: создание токен-ссылки (scope=media|analysis) с настраиваемым TTL (или бессрочно).
- Простой аудит-лог в БД: ключевые события (login, upload, inference, edit, share, export).
- Микросервисная архитектура минимальной сложности (gateway, core-api, inference, postgres, minio).
- Тёмная тема по умолчанию.

### Исключено (вне MVP)
- Уведомления, чат/консультации, многоролевые сложные ACL.
- Версионирование моделей/датасетов, актив-обучение, сегментация/кейпойнты.
- SSO, 2FA, интеграции с LIMS/облаками, образовательный режим.
- Сложные отчёты и дашборды.
- Политики хранения/ретеншн/архивы.

## 5. Функциональные требования
### 5.1 Пользователи и организации
- Регистрация пользователя, создание личного пространства.
- Создание организации (OrgAdmin), приглашение участников (упрощённо — ввод email и роль).
- Лимит seats хранится, но не валидируется тарификацией (MVP).

### 5.2 Классы
- Отображение **глобальных** базовых классов и **org-scoped** пользовательских.
- CRUD пользовательских классов (OrgAdmin/Member).
- При смене класса у детекции:
  - сохранять `orig_class` (от модели) и текущее `class` (после правки).

### 5.3 Медиа-поток
- `POST /media` → presigned PUT → загрузка в MinIO → фиксация метаданных.
- Автогенерация превью (thumbnail) для изображений.

### 5.4 Инференс
- Синхронный вызов сервиса инференса для указанного media_id.
- Параметры видео по умолчанию: `sample_fps=1`, `max_frames=10` (переопределяемые запросом).
- Результаты сохраняются как записи детекций; для видео — c `frame_index` и `frame_ts_ms`.

### 5.5 Просмотр и правки
- Просмотр медиа с наложением bbox; для видео — выбор кадра.
- Смена класса для конкретной детекции (из селектора классов), мгновенная запись.
- Фильтр/подсветка по классам.

### 5.6 Анализы
- Создать «Анализ», добавить список медиа.
- Пересчитать сводку: количество детекций по текущему `class` (после правок).
- Экспорт CSV: агрегат (`class_name,count`) и, при `?details=1`, детализация по детекциям.

### 5.7 Шаринг
- Создание share-ссылки: область (media|analysis), TTL (часы) или бессрочно.
- Публичный read-only экран по токену; недоступность после истечения TTL.
- Аудит события `share.created`, `share.viewed`.

### 5.8 Аудит
- Фиксация событий: `user.login`, `media.upload_init|done`, `inference.run`, `detection.edited`,
  `analysis.created`, `analysis.export_csv`, `share.created`, `share.viewed`.

## 6. Нефункциональные требования
- **Производительность:** целевое время инференса изображения на CPU ≤ 1 с при input size (например, 640).  
  Для видео — суммарно ≤ 10 × 1 с (ориентир), допускается прогресс-индикатор.
- **Надёжность:** автосоздание buckets/миграций; устойчивость к повторной отправке запросов (idempotency на уровне инференса по media_id допускается по флагу `overwrite`).
- **Масштабируемость:** сервисы разнесены; инференс можно горизонтально масштабировать позже.
- **Безопасность:** Argon2, JWT, CORS, ограничение размера загрузок (≤ 20 MB), валидация MIME.
- **Конфиденциальность:** персональные/PHI данные не хранятся; метаданные сугубо технические/пользовательские.
- **Логи и миграции:** Alembic; структурированные логи (json) на уровне API.
- **Деплой:** Docker Compose; целевая ОС — Linux (x86_64).
- **Тема:** тёмная (доступность не критична в MVP, но базовая контрастность соблюдается).

## 7. Технологический стек
- **Frontend:** React/Next.js (SPA/MPA по выбору), TypeScript.
- **Core API:** FastAPI (Python 3.11), SQLAlchemy, Pydantic, Alembic.
- **Inference:** FastAPI + ONNX Runtime (CPU, возможен OpenVINO EP), YOLOv11 ONNX, NMS на стороне сервиса.
- **Хранилища:** Postgres 16; MinIO (S3-совместимое).
- **Gateway:** Nginx (reverse proxy, статика, лимиты).
- **CI/CD:** базовый (lint/tests/build/docker), в MVP может быть полу-ручной.

## 8. Данные и схема (кратко)
**Основные сущности:** User, Organization, Membership, Class, MediaAsset, Detection, Analysis, AnalysisItem, ShareLink, AuditLog.  
Для видео `Detection` включает `frame_index`/`frame_ts_ms`.

## 9. API верхнего уровня (контуры)
- Auth: `/auth/register`, `/auth/login`
- Orgs/Members: `/orgs`, `/orgs/{id}/members`
- Classes: `/classes` (GET all, POST create org-scoped)
- Media: `/media` (init+presigned), `/media/{id}`, `/media/{id}/detections`
- Inference: `/inference?media_id=...&sample_fps=1&max_frames=10`
- Detections: `PATCH /detections/{id}` (смена класса)
- Analyses: `/analyses`, `/analyses/{id}/summary`, `/analyses/{id}/export.csv`
- Share: `/share` (создать), `/share/{token}` (public view)
- Audit: `/audit` (OrgAdmin)

## 10. UX-потоки
1) **Загрузка → инференс → правки:** пользователь загружает медиа, запускает инференс, при необходимости меняет классы.
2) **Формирование Анализа:** собирает медиа в «Анализ», смотрит сводку, экспортирует CSV.
3) **Шаринг:** создаёт ссылку на «Анализ» или медиа, задаёт TTL, отправляет адресату.

## 11. Критерии приёмки (acceptance)
- Регистрация/логин работают; JWT выдается и проверяется.
- Загрузка файлов ≤ 20 MB в MinIO через presigned URL; карточка медиа отображается.
- Инференс:
  - изображение: запись детекций создаётся; среднее время для тестового набора укладывается в целевой SLA на референс-сервере;
  - видео: не более 10 сэмплированных кадров; детекции привязаны к кадрам.
- Просмотрщик показывает bbox; смена класса сохраняется и видна при повторном открытии.
- Анализ создаётся; сводка по классам корректна; CSV соответствует сводке.
- Share-ссылка открывает read-only представление; закрывается по TTL.
- Аудит-события пишутся в БД для перечисленных действий.
- БД-миграции поднимают чистую инсталляцию без ручных правок.

## 12. Риски и допущения
- **CPU-инференс** может превышать 1 с для больших изображений → смягчается ресайзом к фиксированному input size (например, 640) и INT8-квантизацией.
- **Видео-кодеки**/вмешательство ffmpeg/opencv могут влиять на совместимость → в MVP поддерживаем MP4 (H.264) как референс; AVI «best effort».
- **Пользовательские классы** не версионируются; при переименовании возможны несоответствия в исторических данных (принимается в MVP).
- **Без биллинга**: учёт seats декларативный, не блокирующий.

## 13. План релиза (milestones)
- **M1 (инфраструктура):** Docker-Compose, миграции, MinIO buckets, базовые классы (seed).
- **M2 (ядро):** аутентификация, CRUD медиа, presigned upload, просмотрщик изображений.
- **M3 (инференс):** сервис инференса (изображение+видео-сэмплинг), сохранение детекций.
- **M4 (правки и анализы):** смена класса, создание Анализа, сводка, экспорт CSV.
- **M5 (шаринг и аудит):** share-ссылки с TTL, публичный просмотр, аудит-лог.
- **M6 (полировка):** тёмная тема, стабильность, документация API, скрипты развёртывания.
